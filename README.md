# ANAMNESIS — handoff repo *(working title)*

This is the complete handoff for building the site with **Claude Code**. You drive Claude Code; it reads `CLAUDE.md` automatically and follows the docs. You won't be writing site copy by hand — the prompt-pack does that.

## What's in here
- **`CLAUDE.md`** — the project brain Claude Code loads every session (guardrails, stack, build sequence). *The most important file.*
- **`RUNBOOK.md`** — the step-by-step session guide with copy-paste starter prompts. *Start here after setup.*
- **`docs/`** — architecture, build/motion handoff, copy inventory, the dossier, design references.
- **`research/`** — the banked **facts** that feed all written content.
- **`content-demo/`** — three **finished** symbol entries to build the UI against (real content).
- **`prompts/`** — the copy-generation prompt-pack (Claude writes the site copy from these).
- **`assets/plates/`** — seed public-domain imagery (duotone-ready).

## The website AND the writing live in this one repo
The **site** is built from `docs/`. The **written content** is generated by `prompts/` from `research/` into Astro content files under `src/content/`. Same repo, two jobs, one source of truth.

## How to start
1. Put this folder on your PC. In it: `git init`, commit, and push to a new GitHub repo.
2. Open **Claude Code** in this folder.
3. Open **`RUNBOOK.md`** and run the phases in order, pasting each phase's starter prompt.

Claude Code reads `CLAUDE.md` on its own — you never need to paste the guardrails or context manually.

## Order of work (why it's sequenced this way)
**Seed → frame → fill → expand.** The three `content-demo/` entries are the seed; Claude Code builds the shell *against them* so the layout is designed for real content (not lorem); then the prompt-pack mass-generates the rest of the copy *into* that shell; then you expand to the full roster. You hand-write almost nothing.
